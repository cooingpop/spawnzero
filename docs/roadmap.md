# Roadmap

## Phase 0: Bootstrap landing page

Create the minimal public web base and repository structure.

## Phase 1: Local self-grow script

Use local Ollama/Qwen to inspect repository context and propose one small next change.

## Phase 2: PR generation

Commit generated changes to a new branch and open a pull request.

## Phase 3: Optional auto-merge after CI

Keep auto-merge disabled by default. Allow it only when explicitly enabled and after CI passes.

## Phase 4: Public experiment log

Record what the model proposed, what passed validation, and what humans chose to merge.

## Phase 5: Let the project direction emerge from AI proposals

Avoid prescribing a product direction. Observe how the project evolves from model-generated pull requests.
